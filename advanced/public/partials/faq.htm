About GitAnonymizer

GitAnonymizer is a GitHub anonymization platform for double-blind review workflows. It creates and serves anonymized repository and pull request views, removes identity leaks, and keeps content updated on a configurable schedule.

Origin and Credit

This project is inspired by and built on ideas from the original anonymous4science work (anonymous_github). Credit goes to that project for pioneering practical anonymized sharing for research submissions.

Why We Built This Version

During peak conference submission windows, shared public instances can become overloaded or unavailable. We developed GitAnonymizer to reduce manual workload for authors and chairs by providing a stable, self-managed deployment that stays responsive when deadlines approach.

What the Tool Does

  1. Connects to GitHub with OAuth and reads repository or PR source content.
  2. Applies anonymization rules to text content and metadata.
  3. Publishes anonymous links for repositories (/r/<id>) and PRs (/pr/<id>).
  4. Supports expiration policies, updates, and controlled visibility options.
  5. Provides dashboard controls for refresh, removal, and monitoring.

Improvements in GitAnonymizer

  • Unified dashboard for repository and pull request management in one place.
  • Simplified top navigation and faster access to anonymization actions.
  • One-click copy buttons for generated anonymous links.
  • Auto-generated IDs with optional manual override for controlled sharing.
  • Cleaner default login flow for GitHub OAuth without extra setup in normal use.
  • Operational focus for peak periods to reduce submission-time friction and support load.

Operational Notes

GitAnonymizer is intended to improve reliability and usability, but anonymization quality still depends on masking terms and configuration. Always review generated anonymous links before final submission.
